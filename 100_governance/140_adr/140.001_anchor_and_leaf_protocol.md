# 140.001 ADR: Anchor and Leaf Protocol

## 1.0 Context
During the initial **Genesis Phase (v0.1.x)**, the repository utilized expressive, non-prefixed naming conventions. This created two critical risks:
1.  **Non-Determinism:** Automation and agentic pilots lacked a predictable pattern for sorting and traversing the hierarchy.
2.  **Path Instability:** Expressive naming led to inconsistent path lengths, risking OS-level buffer overflows in deep recursive environments.

## 2.0 Decision
We have implemented the **T3S-130 Anchor and Leaf Protocol**.
* **Action:** All folders and files now lead with a 3-digit recursive index.
* **Action:** All root documents for a block MUST use the `.000` suffix to act as the segment anchor.
* **Action:** Folder names have been condensed to minimize path-length risk while preserving technical distinction.

## 3.0 Consequences
* **Positive:** Absolute sorting predictability; reduced risk of system instability; machine-readable document parity.
* **Negative:** Reduced immediate "narrative" clarity in folder names (mitigated by .000 manifest descriptions).

## 4.0 Verification
This decision satisfies **Assertion A0 (Structural Integrity)** of the Standard of Care.
