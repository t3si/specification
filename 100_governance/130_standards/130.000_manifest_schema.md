# 130 Manifest Schema Standard

## 1.0 Purpose
This standard defines the structural requirements for all normative artifacts within the T3S framework. It ensures documentation is machine-readable, auditable, and deterministically mapped to the system's physical state.

## 2.0 Identity and Naming Convention

### 2.1 The "Anchor and Leaf" Protocol
Filenames must mirror their logical index to ensure global uniqueness and deterministic sorting.
* **Segment Anchors (.000):** The primary document for a folder must use the `.000` suffix. This file serves as the manifest or preamble for that block.
    * *Example:* `130.000_manifest_schema_standard.md`
* **Discrete Leaves (.001-999):** Sub-items within a block append a 3-digit incrementing suffix.
    * *Example:* `130.001_specific_technical_pattern.md`

### 2.2 Recursive Indexing (The Segmented Growth Rule)
* **Fixed-Width:** All index segments must be exactly 3 digits.
* **Deep Nesting:** When a sub-domain requires its own folder hierarchy, a new 3-digit segment is appended via a period. 
    * *Pathing:* `130_patterns/130.100_sub/130.100.000_sub_anchor.md`

### 2.3 Document IDs
* **Format:** `T3S-DOC-[BLOCK]-[INDEX]` (e.g., `T3S-DOC-130-000`).
* **Parity Rule:** The `[BLOCK]-[INDEX]` segments must match the filename's numeric index exactly, substituting dots for hyphens.

## 3.0 Required Metadata (YAML Frontmatter)
All artifacts must contain the following keys. Values are **Strings** unless specified.

| Key | Requirement | Data Type / Enum |
| :--- | :--- | :--- |
| `id` | Required | String (T3S-DOC-XXX-XXX) |
| `urn` | Required | String (See Section 5.0) |
| `title` | Required | String |
| `version` | Required | SemVer String (e.g., `0.2.0-alpha`) |
| `status` | Required | Enum: `Genesis`, `Draft`, `Active`, `Retired` |
| `steward` | Required | String |
| `criticality` | Required | Enum: `Tier-1`, `Tier-2`, `Tier-3` |
| `license` | Required | String (e.g., `CC-BY-4.0`) |
| `copyright` | Required | String |

## 4.0 Semantic Structure
1. **H1 (#):** Document Title and Index.
2. **H2 (##):** Major sections (Purpose, Requirements, etc.).
3. **H3 (###):** Technical assertions or sub-clauses.

## 5.0 Deterministic Linking (The URN Protocol)
* **Regex:** `^urn:t3s:[a-z0-9-]+(?::[a-z0-9-]+)*$`
* **Parity:** URN segments must logically mirror the folder segments. 
    * *File `130.000` -> URN `urn:t3s:pattern:manifest-schema`*

## 6.0 Compliance Gate
Any document failing to meet the following criteria shall return `FALSE` for **Assertion A0 (Structural Integrity)**:
1. **Filename Parity:** Filename must lead with the recursive index (e.g., `130.000_...`).
2. **ID Parity:** YAML `id` must match the filename index (e.g., `130-000`).
3. **Anchor Requirement:** Root files in any directory MUST use the `.000` suffix.
