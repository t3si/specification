# 030 Persona and Identity Manifest

## 1.0 Purpose
This manifest defines the actors within the T3S framework. It categorizes identities into **Human Stewards**, **Agentic Personas** (AI), and **Service Identities** to ensure all system actions are attributable to a verified URN.

## 2.0 Actor Classifications
1. **Human Steward (HS):** The sovereign authority. Responsible for signing ADRs and Mandate updates.
2. **Agentic Persona (AP):** Automated entities (e.g., T.E.S.S.) that operate within the boundaries of the **060 Automation Logic**.
3. **Service Identity (SI):** Non-agentic system processes (e.g., backup scripts) requiring cryptographic identification.

## 3.0 Identity Standards
All personas must be indexed in the **070.000 Root Manifest** and mapped to a cryptographic public key in the **040 Integrity** block.
