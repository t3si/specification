# 050 Core Ontologies and Schemas Manifest

## 1.0 Purpose (ITIL SKMS Alignment)
This manifest governs the **Service Knowledge Management System (SKMS)**. It ensures **Semantic Stewardship (Objective B)** by providing the formal syntax required to transform raw repository data into actionable governance intelligence. It serves as the "Master Schema" for all automated reasoning within the Service Value System.

## 2.0 Semantic Foundations
To prevent "Linguistic Drift" and ensure interoperability between human stewards and AI agents, the following anchors are mandated:

### 2.1 The T3S Controlled Vocabulary
All artifacts must utilize terms defined in the `https://t3si.org/ontology/` namespace:
* **SystemGraph:** The top-level configuration record of the repository state.
* **Steward:** The persona/entity held accountable for the lifecycle of a Configuration Item (CI).
* **MarkovBlanket:** The deterministic boundary of sovereign control and data egress.
* **Assertion:** A verifiable statement of fact mapped to a **Standard of Care (120)** requirement.

### 2.2 Global Schema Mapping
In alignment with ITIL 4 **Information and Technology** guidelines, internal terms are mapped to established global standards:
* `t3s:ServiceConfiguration` ↔ `schema:SoftwareSourceCode`
* `t3s:ChangeRecord` ↔ `schema:UpdateAction`
* `t3s:EvidenceRecord` ↔ `schema:Dataset` (mapped to ITIL Forensic Evidence requirements)

## 3.0 Schema Governance & Change Control
* **Validation:** All JSON-LD and YAML manifests MUST validate against the **130.000 Manifest Schema** before promotion to `Active`.
* **Impact Assessment:** Any modification to core ontologies constitutes a **Major Change** under ITIL guidelines, requiring a **140.xxx ADR** and a full regression test of the `060.000` validation pipelines.

## 4.0 Taxonomic Integrity
The 3-digit indexing system is defined as a **Positional Ontology**. The numeric prefix (e.g., 100, 200) creates an inherited authority chain, ensuring that "Control" (000) always precedes "Strategy" (100) and "Execution" (200+).
